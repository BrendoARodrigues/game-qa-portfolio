# BUG-001 — Aquatic Enemy Stops Attacking When Player Positions Underneath It

| Field          | Details                        |
| -------------- | ------------------------------ |
| ID             | BUG-001                        |
| Game           | Metal Garden                   |
| Game Version   | 2.7.2                          |
| Date Reported  | 2026-06-11                     |
| Reported By    | Brendo de Araujo               |
| Severity       | Medium                         |
| Priority       | Medium                         |
| Status            | Confirmed                         |
| Reproduction Rate | Multiple successful reproductions |
| Category          | AI / Combat Behavior              |
| Area           | Aquatic Zone — Enemy Encounter |

---

## Description

The aquatic enemy ceases its attack behavior when the player positions themselves directly underneath it. Once the player occupies this position, the enemy becomes idle and does not resume attacking until the player moves away.

---

## Steps to Reproduce

1. Enter the aquatic zone and locate an aquatic enemy.
2. Navigate the player character to a position directly below the enemy.
3. Remain stationary in that position.
4. Observe the enemy's response.

---

## Expected Result

The aquatic enemy should maintain its attack pattern and continue engaging the player regardless of the player's vertical position relative to the enemy.

---

## Actual Result

The aquatic enemy stops attacking entirely when the player is positioned directly below it. The AI has no defined response behavior for this positional state and remains idle.

---

## Impact

The player can neutralize the enemy's threat without performing any offensive action, effectively bypassing the encounter by exploiting the AI's positional blind spot.

> Note: The player cannot attack while underwater, there is no XP system, no loot, and no economy mechanic in this area. The exploit impact is limited to trivializing the encounter, with no effect on progression or game economy.

---

## Severity Justification

**Severity: Medium** — The bug disrupts the intended combat design by fully disabling enemy behavior. While no critical system is affected, it undermines encounter integrity and player challenge.

**Priority: Medium** — Should be addressed to preserve the intended combat experience. Does not block progression or cause data loss.
